# КИБЕР · Design Handoff

Мобильная веб-версия карточной настольной игры. Документ описывает актуальный layout для передачи дизайнеру.

**Live:** https://yarnebanan-dotcom.github.io/cyber-game/
**Target viewport:** portrait, 360–480 px ширина · 640–900 px высота
**Testing viewports:** 375×667 (iPhone SE), 360×640 (Android mid-range), 390×844 (iPhone 13)

---

## 1. Концепция экрана

Вся игра живёт на **одном экране** без прокрутки. Все 4 информационные зоны + HUD всегда видны одновременно, их высоты **фиксированы**, поле внутри сжимается/расширяется по остаточному месту.

```
┌─────────────────────────────────┐
│  HUD  (имя · очки · ход · фаза) │  ~90 px
├─────────────────────────────────┤
│  PHASE HINT (оранжевая подсказка)│  28 px  ◀ fixed
├─────────────────────────────────┤
│  РАСКРЫТО (общая зона карт)     │  72 px  ◀ fixed
├─────────────────────────────────┤
│                                 │
│         ИГРОВОЕ ПОЛЕ             │  остаток (1fr)
│                                 │
├─────────────────────────────────┤
│  ОПИСАНИЕ КАРТЫ                 │  72 px  ◀ fixed
├─────────────────────────────────┤
│  РУКА · карты текущего игрока    │  110 px ◀ fixed
├─────────────────────────────────┤
│  ACTION BAR (2 или 3 кнопки)    │  39 px  ◀ fixed
└─────────────────────────────────┘
```

Между секциями — gap **3 px**. Слева/справа — padding **6 px**. Снизу — `safe-area-inset-bottom` (iOS home indicator).

## 3. Фиксированные зоны — подробности

### 3.1 HUD (~90 px)

- Имя игрока (P1/P2/P3) + индикатор цвета (ромб `p1/p2/p3`)
- Счёт (большая цифра справа · `/20` или `/15`)
- Кнопка паузы `≡` (40×40)
- Под ним — мета-строка: `ХОД T01 · ФАЗА ●0/2 · ▶0/2 · ✦0/2 · P2:КОЛО…`
- Стэппер фаз: `01·ВОСПОЛН` → `02·ХОД` → `03·КОНЕЦ` (активная подсвечена accent)

### 3.2 Phase Hint (28 px fixed)

- Оранжевая подсказка в пунктирной рамке accent
- Формат: `> СТАВЬ ФИШКИ · РОЗЫГРЫШ НЕВОЗМОЖЕН`
- Справа — счётчик `●2/2` (фишки)
- Текст mono 10 px, **overflow: hidden** (чтобы перенос текста не двигал поле)

### 3.3 Раскрыто (72 px fixed)

Горизонтальная лента общих «раскрытых» карт (любой игрок может разыграть свою раскрытую карту).

- **Всегда видна**, даже когда карт нет
- Пустое состояние: dashed border, плейсхолдер `— ПУСТО —` (text-ghost 9 px)
- С картами: превью карт 52×58 px, прокрутка горизонтальная
- Заголовок строки: `РАСКРЫТО · ОБЩАЯ ЗОНА · N КАРТ` (mono 10 px, text-dim)

### 3.4 Поле (остаток)

- Квадратное, `width = height = min(width, height)` (через container queries)
- Сетка 4×4 (2p) или 5×5 (3p)
- Beveled corners (clip-path polygon, 14 px срез)
- Узлы (`node`) — ромбы, размер **≥ 41 px** (минимальный tap target на 360×640 3p HARD)
- Фишки — кружки цвета игрока с glow

На 375×667 2p: поле **232×232** (58 px/узел)
На 360×640 3p HARD: поле **205×205** (41 px/узел) — **минимальный кейс**

### 3.5 Описание карты (72 px fixed)

- **Всегда видно**, размер не меняется
- Пустое: dashed border text-ghost, плейсхолдер `— ВЫБЕРИ КАРТУ · ОПИСАНИЕ ЭФФЕКТА —`
- С картой: заголовок (имя + стоимость) + строки эффектов:
  - `▶ РОЗЫГРЫШ · …` (cyan `--line`)
  - `✦ УТИЛИЗАЦИЯ · …` (accent)
  - `⊕ СИНТЕЗ · …` (violet `--p3`)
- Если описание длинное — `overflow-y: auto` внутри слота
- mono 10 px, line-height 1.35

### 3.6 Рука (110 px fixed)

- Заголовок: `○ РУКА · Игрок N · · · K/M` (K текущие, M supply)
- Горизонтальная прокрутка карт
- Карта: **82×88 px**, padding 4 px
  - Cost-chip 16×16 (шрифт 9 px)
  - Паттерн 3×3 ячейки (cyan фишка / grey фишка противника)
  - Имя 8 px, letter-spacing 0.1em
- Selected card: `translateY(-8px)` + glow
- Пустое место **не коллапсится** (высота фиксирована)

### 3.7 Action Bar (39 px fixed)

- 2 кнопки (normal): `✦ УТИЛИЗИРОВАТЬ` · `⏭ ЗАВЕРШИТЬ ХОД`
- 3 кнопки (Hard mode): `✦ УТИЛИЗИРОВАТЬ` · `+3 ДОБОР` · `⏭ ЗАВЕРШИТЬ ХОД`
- Flex, равная ширина, gap 6 px
- `padding: 11px 6px`, mono 11 px
- disabled → opacity 0.35

## 5. Состояния экрана

| Состояние | Описание | Что видно |
|---|---|---|
| **Idle** | Карта не выбрана, раскрытых нет | Все 4 зоны, обе с плейсхолдерами |
| **Card selected** | Игрок тапнул по карте в руке | Desc показывает эффекты; карта приподнята |
| **Revealed present** | В общей зоне лежат карты | Раскрыто с превью (52×58); клик → desc |
| **Hard mode** | Запущен с чекбоксом HARD | Action bar с 3 кнопками (+3 Добор) |
| **Synth panel** | Два Синтез-эффекта, нужен порядок | Между hand и action появляется панель с A/B (влияет на layout? нет — помещается в 1fr остаток) |

## 6. Интерактив и обратная связь

- Тап на карту → выбор (glitch + glow)
- Двойной тап на карту → поворот паттерна (90°)
- Долгое удержание → zoom карты (modal, z 1300)
- Тап по свободному узлу → постановка фишки (с haptic `vibrate(10)`)
- Тап по узлу с фишкой → снятие (только в фазу Действий, если ещё не зафиксирована)
- Handoff screen между ходами игроков (glitch-анимация, кнопка `ПЕРЕДАТЬ →`)

## 8. Известные ограничения / вопросы к дизайнеру

- Поле на 360×640 3p HARD — **41 px/узел**. Это минимум для tap-target. Можно ли резать вертикальный бюджет ещё — например, объединить hint + HUD?
- Описание карты в 72 px — длинные тексты скроллятся. Стоит ли увеличить до 90 px за счёт руки (с карт 82→72 px)?
- Раскрытая лента 72 px — карты там 52×58 px, читаемость низкая. Увеличить высоту?
- Рука: при 5+ картах идёт горизонтальный скролл. Подсветить scroll-hint?
- Индикатор фишек противника (grey) в паттерне карты — отличим ли от своего при тусклом экране?

---

## 9. Что НЕ надо трогать

- **Баланс и механика** — числа, эффекты карт, паттерны зафиксированы (54 карты). Менять нельзя.
- **Цвета игроков** — P1 cyan / P2 orange / P3 violet — канон проекта.
- **9-row grid** — структура layout. Можно менять высоты, нельзя менять порядок зон.
- **Fixed heights** — любая зона должна иметь explicit `height` + `box-sizing: border-box`, иначе поле будет дёргаться при смене состояния.
